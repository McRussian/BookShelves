# BookShelves — Анализ концепции

## 1. Суть продукта

Десктопное приложение для управления домашней электронной библиотекой в локальной сети.

**Что делает:**
- Хранит метаданные книг в общей SQLite-базе (на сетевой папке / NAS)
- Управляет файлами книг на локальном устройстве или сетевом диске
- Позволяет каждому пользователю создавать личные полки из общей библиотеки
- Привязывает форматы файлов к программам-ридерам (настройка хранится локально на каждом устройстве)

**Что не делает (за рамками текущего скоупа):**
- Нет публикации в интернет
- Нет облачного хранения
- Нет социальных функций (публичные полки, рекомендации)
- Нет площадки для самиздата

## 2. Архитектура

```
Устройство 1 (ПК)          Устройство 2 (ноутбук)
┌─────────────────┐         ┌─────────────────┐
│  PyQt5-клиент   │         │  PyQt5-клиент   │
│  QSettings      │         │  QSettings      │
│  (локальные     │         │  (локальные     │
│   настройки)    │         │   настройки)    │
└────────┬────────┘         └────────┬────────┘
         │                           │
         └──────────┬────────────────┘
                    │ Peewee ORM (прямое подключение)
                    ▼
         ┌─────────────────┐
         │  SQLite (.db)   │  ← сетевая папка / NAS
         └─────────────────┘
```

**Локальные настройки каждого клиента (QSettings, не в БД):**
- Путь к файлу базы данных
- Привязки форматов к ридерам (`epub → Foliate`, `pdf → Evince` и т.п.)
- Настройки интерфейса

**Почему SQLite на сетевой папке — допустимо:**
Операции записи (добавление, редактирование книг) редки и происходят преимущественно на этапе начального заполнения. Основная нагрузка — чтение. SQLite в WAL-режиме справляется с таким паттерном без риска повреждения данных.

## 3. Модель данных

```
Общие для всех пользователей:
  Author      — авторы
  Genre       — справочник жанров (фиксированный список)
  Tag         — теги (свободный ввод)
  BookFormat  — форматы файлов (epub, pdf, fb2, mobi, djvu...)
  Book        — книги
  BookAuthor  — книга ↔ автор       (многие-ко-многим)
  BookGenre   — книга ↔ жанр        (многие-ко-многим)
  BookTag     — книга ↔ тег         (многие-ко-многим)

Личные (у каждого пользователя своё):
  User        — пользователи системы
  Shelf       — полка (принадлежит пользователю)
  ShelfBook   — полка ↔ книга       (многие-ко-многим)
```

**Ключевые решения по модели:**
- Жанры — фиксированный справочник, не свободный ввод
- Тип книги — формат файла (для привязки к ридеру), хранится в `BookFormat`
- Одна книга может лежать на нескольких полках одновременно
- Полки у каждого пользователя свои, библиотека общая

## 4. Востребованность

### Подтверждённый спрос

Calibre — главный аналог в сегменте локального управления книгами — насчитывает
более **30 млн загрузок** и ~25 000 участников в r/Calibre. Это прямое подтверждение:
миллионы людей хотят управлять книгами локально.

### Незакрытые потребности Calibre

Calibre решает задачу хранения, но имеет хронические слабые места:

| Потребность | Calibre | BookShelves |
|---|---|---|
| Несколько переключаемых полок | ❌ только одна библиотека | ✅ |
| Быстрое переключение между полками | ❌ | ✅ |
| Динамические полки (по правилам) | ❌ только поиск/теги | ✅ |
| Личные полки у каждого пользователя | ❌ | ✅ |
| Современный интерфейс | ❌ устаревший UI | ✅ |
| Сетевой доступ для нескольких устройств | ⚠️ сложная настройка | ✅ |

### Целевая аудитория

- Домашние пользователи с большой коллекцией электронных книг
- Семьи или небольшие группы, читающие из общей библиотеки
- Владельцы NAS и домашних серверов, предпочитающие локальные решения

## 5. Сравнение с аналогами

### Calibre
**Плюсы:** зрелый продукт, конвертация форматов, плагины, бесплатный.
**Минусы:** одна библиотека без переключаемых полок, нет многопользовательского режима, устаревший интерфейс, мобильный доступ требует сложной настройки.

### Alfa EBooks Manager
**Плюсы:** локальное хранение, статистика чтения.
**Минусы:** только Windows, нет сетевого режима, нет многопользовательского доступа.

### Calibre-Web
**Плюсы:** веб-интерфейс поверх библиотеки Calibre, мобильный доступ.
**Минусы:** требует отдельного сервера, сложная настройка, наследует ограничения Calibre по структуре полок.

### Итог

Ни один из существующих инструментов не сочетает одновременно:
- локальное хранение без сервера
- несколько личных полок с быстрым переключением
- многопользовательский режим в локальной сети
- простую настройку и современный интерфейс

## 7. Риски

| Риск | Уровень | Снижение |
|---|---|---|
| Конкуренция с Calibre (зрелый продукт) | Средний | Фокус на UX и многопользовательских полках |
| Одновременная запись в SQLite по сети | Низкий | WAL-режим; записи редки по характеру использования |
| Сложность настройки сетевого пути для обычных пользователей | Средний | Мастер первоначальной настройки при первом запуске |
